Installation

Available as either static or dynamic library.

Flesto

Instead of keeping your data in one continuous array, which you will probably need to shrink and grow, manage multiple arrays as if they were one. Add and remove them whenever you want.

Mufex

Coming soon! Mutual function exclusion. Specify which functions can run at the same time. No need to rewrite your code to use mutexes.

Time

Coming soon! API to create timeouts, similarly to JavaScript. Without kernel calls[1], making code slower. Without endless threads for each timeout - only one for all.
[1] except calls fetching current time

AVL tree

Coming soon! Highly optimised AVL tree, compatible with flesto, capable of holding any data using pointers. You only need to store it somewhere, leave the rest to the tree.

shnet

Coming soon! This covers everything else not covered above. Create connections, manage them using high performance epoll and barely any threads. Tweak it to suit exactly what you need. No delaying using event loop - you instantly get notified if anything happens. No wack documentation - you exactly know which function returns what and can adapt your code to perfectly handle errors. Created with extensibility in mind - adding new protocols, such as QUIC, should not be a big deal.
